# data/employee_generation_inputs/

Source prior files used by `novawireless-rep-gen` to seed call center representative persona generation.

These files are **inputs**, not outputs. They are not generated by the pipeline — they are calibration sources that shape the distribution of rep traits, skill specializations, and workforce pressure parameters.

---

## Files

| File | Source | Used For |
|---|---|---|
| `kaggle_employee_persona_priors.csv` | Kaggle call center workforce dataset | Patience, empathy, escalation proneness, burnout risk distributions |
| `fcc_cgb_consumer_complaints__rep_specialization_priors.csv` | FCC Consumer & Governmental Affairs Bureau complaint data | Skill tag distribution (billing, device, network, porting, fraud) |
| `kaggle_call_center_weekday_pressure.csv` | Kaggle call center operations dataset | Weekday pressure index — shapes per-call friction tiers |
| `ibm_telco_segment_pressure.csv` | IBM Telco customer churn dataset | Segment-level pressure index — adjusts baseline workforce strain |

---

## How These Are Used

`generate_employees_call_center_one_queue.py` loads these files at runtime and uses them to:

1. Sample persona traits (patience, empathy, escalation proneness, burnout risk) weighted by the kaggle persona priors
2. Assign primary and secondary skill tags weighted by FCC complaint category frequencies
3. Compute a pressure index from the weekday and telco segment files that feeds into burnout and compliance risk synthesis

If any of these files are missing, the generator falls back to hardcoded defaults and continues without error.

---

## Notes

- These files contain aggregated, anonymized prior distributions — not individual records.
- They do not contain any real employee data.
- Modifying these files will change the distribution of rep traits in generated employee databases.
